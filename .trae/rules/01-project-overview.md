# Project Overview

## 项目定位
这是一个面向“重要任务启动困难 / 拖延 / 焦虑 / 回避”的任务管理 Web App。

它不是普通 Todo App。核心目标不是单纯记录任务，而是帮助用户：
1. 识别任务阻力
2. 降低启动门槛
3. 把高压任务转化为可开始的最小动作
4. 奖励“面对”和“开始”，而不仅是“完成”
5. 记录回避模式，减少内耗和羞耻感

一句话定义：
“不是帮用户记住任务，而是帮用户开始任务。”

## 要解决的问题
- 明明知道任务重要，但本能回避
- 不一定难，而是心理上难以启动
- 越重要越拖延
- 拖延后焦虑、羞耻、自责加重
- 传统任务工具只会施压提醒，无法解决启动阻力

## 项目边界
本项目不是：
- 团队协作工具
- OKR / Jira / 排期系统
- 社交打卡产品
- 聊天陪伴型 AI 产品
- 重度日历系统
- 复杂游戏化平台

MVP 阶段禁止扩展成复杂项目管理系统。

## 核心原则
1. 优先做可运行 MVP
2. 允许用户“不完美启动”
3. 首页优先回答“今天怎么更容易开始”
4. 奖励面对、接触、微启动、重启
5. 所有设计都围绕“降低启动阻力”

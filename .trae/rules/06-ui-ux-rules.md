# UI UX Rules

## 总体风格
这是一个减压型任务工具，不是高压效率面板。

UI 必须满足：
1. 简洁
2. 柔和
3. 安静
4. 信息不过载
5. 鼓励启动而不是施压

## 页面体验原则
1. 首页优先展示“今天怎么更容易开始”
2. 不要首页堆满未完成任务形成压迫感
3. 高恐惧任务要被识别，但不能像惩罚标记
4. contactStep 和 tinyStep 必须视觉突出
5. 状态、积分、复盘都应服务于“降低启动门槛”

## 视觉约束
- 避免大量红色警报式提示
- 避免复杂高密度仪表盘
- 避免把高阻力任务做成负面排行榜
- 要有友好空状态和引导提示

## 文案风格
优先使用减压、允许启动困难的语气，例如：
- 今天不要求完成，只要求接触
- 先做一个最小动作就够了
- 回避被记录下来，不代表失败
- 看见阻力，本身就是进展

## 交互原则
1. 能少一步就少一步
2. 编辑阻力信息不能太重问卷化
3. 打卡要轻量快速
4. 用户应能快速看到“下一步最小动作”
5. 不要让提醒本身成为新的压力

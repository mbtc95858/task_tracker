# Delivery And Acceptance

## 阶段性输出要求
每个阶段结束后都必须说明：
1. 已完成内容
2. 当前项目结构
3. 下一步计划

## 最终交付必须包含
1. 本地运行方式
2. 数据库初始化方式
3. seed 执行方式
4. 页面说明
5. 第二版扩展建议

## MVP 验收标准
最终必须满足：
1. 可以创建、编辑、删除任务
2. 任务可保存阻力分析字段
3. 任务可设置三层动作
4. AVOIDED 状态可切换
5. Progress Log 可记录
6. 积分由后端正确结算
7. Dashboard 展示真实任务与推荐动作
8. Daily Review 可填写并保存
9. Insights 有基础统计
10. 项目可本地完整运行演示

## 开发纪律
1. 不允许只做静态假页面
2. 不允许跳过 schema / API / 数据库
3. 不允许为了快而堆硬编码
4. 不允许先做花哨 UI 再补核心链路
5. 优先保证功能真实可跑通

## 成功标准
这个 MVP 的成功，不是页面多，而是：
- 用户能更容易面对重要任务
- 用户能记录阻力
- 用户能得到最小启动动作
- 用户能在回避后重新启动

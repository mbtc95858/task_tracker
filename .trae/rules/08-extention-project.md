# Project Rule Extension — Project / Tree Task / Board

本项目在原有 Task 系统基础上，新增“长任务（Project）+ 树状任务结构 + 横向任务看板”能力。

## 1. 核心目标（必须遵守）

本模块的目标不是做项目管理工具，而是：

帮助用户把“难以启动的长任务”拆解为可开始、可推进、可聚焦的路径。

优先解决：
- 拖延
- 启动困难
- 任务过大带来的心理阻力

而不是：
- 团队协作
- 复杂项目管理
- 甘特图 / 资源调度

---

## 2. 数据结构原则

1. Task 必须支持树状结构（parentTaskId）
2. Project 是 Task 的容器（不是替代 Task）
3. 不允许破坏原有 Task 行为
4. 所有新增字段必须向后兼容

必须存在：
- Project
- Task.parentTaskId
- taskType（阶段 / 任务 / 子任务）

---

## 3. 进度原则

支持两种模式：

1. AUTO（默认）
- 根据子任务状态自动计算
- 优先简单可运行

2. MANUAL
- 手动填写

核心要求：
- Project 必须有整体进度
- 节点必须有局部进度

---

## 4. Project Board（关键模块）

必须满足：

1. 横向树状结构（从左到右）
2. 基于真实数据（禁止静态 mock）
3. 支持展开 / 折叠
4. 支持子任务新增
5. 支持状态切换
6. 支持节点详情查看

禁止实现：
- Kanban
- 甘特图
- 自由拖拽画布
- 任意连线图编辑器

---

## 5. 高阻力节点（必须保留）

节点满足以下任一条件即高阻力：
- fearLevel >= 7
- resistanceLevel >= 7
- startDifficulty >= 7
- status = AVOIDED

UI 必须提供：
- 轻量高亮（非警报式）
- 可快速查看原因
- 可直接触发 tinyStep / contactStep

---

## 6. 推荐路径（核心差异）

系统必须能够推导：

“当前最值得开始的节点”

规则优先级：
1. 未完成
2. 高重要度
3. AVOIDED → contactStep
4. 高阻力 → tinyStep
5. 已启动 → normalStep

---

## 7. UX 原则（非常重要）

1. 默认不全展开（避免压迫感）
2. 强调“当前一步”，不是“全局复杂性”
3. 信息密度适中
4. 详情放抽屉
5. 页面必须回答：
   - 现在做什么？
   - 最容易开始的是哪个？
   - 哪个节点卡住了？

---

## 8. 实现优先级（必须遵守）

1. Project CRUD
2. Task 树结构
3. 进度计算
4. Project Detail
5. Project Board（核心）
6. 高阻力 + 推荐路径

禁止跳过核心链路直接做 UI 优化
